# Contributing

## Development

Use the Go version from `go.mod`.

Run before opening a PR:

```bash
make standards-check TOPIC="short-topic-for-change"
make decision-contract-check
make fmt
make vet
make lint
make test
```

## Release Notes

Release notes are generated by CI from tags/changelog script.

## Rules

- Keep changes minimal, idempotent, and fail-fast.
- Avoid environment-specific hardcoded values.
- Add tests for behavior and validation paths.
- Treat `docs/decisions` as normative: if behavior/flow changes and no accepted decision exists, create/update decision first.
- For behavior/UX changes in wizard flows, request explicit approval before implementation.
- Never leave sensitive plaintext in `*.sops.yaml`; use shared SOPS helpers only.
